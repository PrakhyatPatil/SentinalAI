# SafeRoute 🛡️

### *IntelliAI Arena 2026 · Track 2, Problem Statement 3*
**SafeRoute** is a premium, AI-powered women's safety navigation platform. It enables users to plan routes, analyze risk in real-time, view crowd-reported incident heatmaps, and obtain natural-language safety briefings powered by Gemini AI.

---

## 🌟 Key Features

### 1. Immersive Map-Centric UI
- Full-viewport interactive map with floating glassmorphism overlays, heavy backdrop blur (`24px`), and elegant card shadows.
- Modern interface aesthetics inspired by Apple, Stripe, and Linear.

### 2. Time-Aware Incident Heatmaps
- Interactive heatmap visualising crowd-reported safety incidents.
- **Time-of-day slider (0–23h):** Adjusts incident weights dynamically in the browser (e.g., higher severity weights for poor lighting and isolation after 9 PM).

### 3. Smart Route Scoring & Color-Coding
- Queries the Google Maps Directions API for pedestrian routes.
- Cross-references route waypoints against Firestore incidents using **Haversine distance (150m radius)**.
- Segment-level risk calculation, rendering the path in **Green** (Safe), **Amber** (Caution), or **Red** (High Risk).
- Displays an aggregate safety score badge (0–100).

### 4. Gemini AI Safety Summaries
- Feeds route waypoints, nearby hazard listings, and computed scores to **Gemini 1.5 Flash**.
- Displays a calm, actionable 2–3 sentence natural-language safety brief directly in the side panel.
- Built-in graceful degradation: falls back to locally generated safety briefings if Gemini is unreachable.

### 5. Pulsing One-Tap SOS FAB
- Prominent pulsing SOS button positioned in the header on desktop and as a floating action button (FAB) on mobile.
- Dynamically shifts vertical position on mobile to remain visible above the sliding bottom sheet.
- Fetches live device GPS coordinates and pre-fills a WhatsApp message: `wa.me/?text=SOS...` deep link.

### 6. Frictionless Access (Silent Auth)
- Utilises **Firebase Anonymous Authentication** to sign in users silently on page load, eliminating signup friction.

---

## 🛠️ Technology Stack

- **Frontend Framework:** React 18 + Vite (SPA)
- **Map & Geocoding Services:** Google Maps JavaScript SDK v3 (`@react-google-maps/api`)
- **Database & Auth:** Firebase Firestore & Firebase Anonymous Auth
- **AI Engine:** Google Gemini 1.5 Flash (`@google/generative-ai`)
- **Styling:** Vanilla CSS (custom design system)

## 🚀 Quick Start & Setup Guide

Ensure you have [Node.js](https://nodejs.org/) installed.

### 1. Clone the Repository
```bash
git clone https://github.com/PrakhyatPatil/wayward-ai.git
cd wayward-ai
```

### 2. Install Dependencies
```bash
npm install
```

### 3. Configure Environment Variables
Create a `.env.local` file in the root directory. Configure the following variables using your credentials:

```env
VITE_GOOGLE_MAPS_API_KEY=your_google_maps_api_key
VITE_GEMINI_API_KEY=your_gemini_api_key
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

> [!NOTE]
> Make sure to enable **Maps JavaScript API**, **Directions API**, and **Places API** in your Google Cloud Console for the Maps key.

### 4. Run the Development Server
```bash
npm run dev
```
Open `http://localhost:5173` in your browser.

### 5. Build for Production
```bash
npm run build
```

## 🏁 Judge's Demo Walkthrough

Walk through these steps to evaluate the SafeRoute application in 2 minutes:

1. **Initial View:** Open the app. Observe the pre-seeded safety incident hotspots centered around Rajwada / Vijay Nagar / AITR campus in Indore.
2. **Time-of-day Weighting:** Move the time slider at the bottom of the sidebar to **11:00 PM**. Observe the heatmap intensity glowing and expanding to reflect increased late-night risk.
3. **Plan a Route:** Click **"Load Demo Route"** to autofill *AITR College* to *Vijay Nagar Square*. Click **"Find Safe Route"**.
4. **Scoring & Path:** Notice the route polyline rendered on the map. Red and amber segments represent proximity to reported incidents. The sidebar displays a safety verdict card (e.g., `74/100 Moderate`).
5. **AI Safety brief:** See the **AI Safety Summary** panel load the brief. Read the plain-language safety guidelines generated by Gemini.
6. **Report an Incident:** Click anywhere on the map to open the incident reporter popup. Select "Poor Lighting" or "Harassment History" and click **Submit**. The route segments re-calculate and re-color dynamically.
7. **Emergency SOS:** Click the **SOS button**. Grant location permission to see it fetch coordinates and generate a WhatsApp share draft with your live coordinates.
